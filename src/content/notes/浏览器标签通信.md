# 浏览器标签页通信

浏览器标签页之间有多种通信方式，各有适用场景。

## 1. postMessage

如果能获得对应标签页的引用，可以使用 `postMessage` 进行跨窗口通信。

```
otherWindow.postMessage(message, targetOrigin, [transfer]);
```

- **otherWindow**：其他窗口的引用（通过 `window.open`、iframe 等获取）
- **message**：发送到其他窗口的数据，无需自己序列化
- **targetOrigin**：指定哪些窗口能够接收到消息（设为具体域名，更安全）
- **transfer**（可选）：和 message 同时传递的 transferable 对象

### 示例： opener 与 popup 通信

```html
<!-- A 窗口 (http://example.com:8080) -->
<script>
  var popup = window.open('http://example.org');

  // 发送消息到 popup
  popup.postMessage('hello there!', 'http://example.org');

  // 监听 popup 的回复
  function receiveMessage(event) {
    // 始终验证来源
    if (event.origin !== 'http://example.org') return;

    console.log('收到回复:', event.data);
    // event.source 是 popup 的引用
  }
  window.addEventListener('message', receiveMessage, false);
</script>
```

```html
<!-- popup 窗口 (http://example.org) -->
<script>
  function receiveMessage(event) {
    // 验证来源
    if (event.origin !== 'http://example.com:8080') return;

    console.log('收到消息:', event.data);

    // 回复 opener，使用 event.origin 作为 targetOrigin
    event.source.postMessage('hi there yourself!', event.origin);
  }
  window.addEventListener('message', receiveMessage, false);
</script>
```

### 安全问题

- 如果不需要从其他网站接收 message，不要为 message 事件添加任何监听器
- 始终使用 `origin` 和 `source` 属性验证发件人的身份

## 2. SharedWorker（边缘）

`SharedWorker` 会在页面存在的生命周期内创建一个唯一的线程，多个页面共享同一个线程，适合需要长时间保持状态的场景。

```js
// shared-worker.js
const connections = new Set();

self.onconnect = (event) => {
  const port = event.ports[0];
  connections.add(port);

  port.onmessage = (e) => {
    // 广播给所有连接的页面
    connections.forEach((p) => {
      if (p !== port) p.postMessage(e.data);
    });
  };

  port.start();
};
```

```js
// 页面 A
const worker = new SharedWorker('/shared-worker.js');
worker.port.onmessage = (e) => console.log('收到:', e.data);
worker.port.start();
worker.port.postMessage('hello from A');
```

```js
// 页面 B（同样代码）
const worker = new SharedWorker('/shared-worker.js');
worker.port.onmessage = (e) => console.log('收到:', e.data);
worker.port.start();
worker.port.postMessage('hello from B');
```

## 3. BroadcastChannel

同源标签页之间通信的轻量方式，比 postMessage 更简单，不需要持有窗口引用。

```js
// 创建频道
const channel = new BroadcastChannel('my_channel');

// 发送消息
channel.postMessage('hello');

// 接收消息
channel.onmessage = (event) => {
  console.log('收到:', event.data);
};

// 关闭频道
channel.close();
```

特点：
- 同源即可通信（http://example.com 和 http://example.com:8080 **不通**）
- 所有标签页共享同一个频道名
- 适合标签页间的实时同步、状态通知

## 4. localStorage

对 `localStorage` 进行监听，当一个页面修改数据时，另一个页面会触发 `storage` 事件：

```js
window.addEventListener('storage', (e) => {
  console.log('key:', e.key, 'newValue:', e.newValue);
});
```

注意：`storage` 事件只在**其他**标签页触发，当前标签页不会收到。

## 5. WebSocket

WebSocket 协议可以实现服务器推送，服务器作为中介者转发消息。适合需要实时双向通信的场景。

```js
// 客户端
const ws = new WebSocket('wss://example.com/ws');

ws.onopen = () => ws.send('hello');

ws.onmessage = (event) => {
  console.log('收到:', event.data);
};
```

```js
// 服务器端 (Node.js ws 示例)
const { WebSocketServer } = require('ws');
const wss = new WebSocketServer({ port: 8080 });

wss.on('connection', (ws) => {
  ws.on('message', (data) => {
    // 广播给所有客户端
    wss.clients.forEach((client) => {
      if (client !== ws && client.readyState === 1) {
        client.send(data);
      }
    });
  });
});
```

## 通信方式对比

| 方式 | 实时性 | 跨域 | 复杂度 | 适用场景 |
|------|--------|------|--------|----------|
| postMessage | 即时 | 支持（指定 targetOrigin） | 低 | 两个窗口互相引用 |
| SharedWorker | 即时 | 支持 | 中 | 多页面共享状态 |
| BroadcastChannel | 即时 | 同源 | 低 | 同源多标签页同步 |
| localStorage | 稍延迟 | 同源 | 低 | 同源页面同步 |
| WebSocket | 即时 | 支持 | 高 | 需要服务中介、实时推送 |
